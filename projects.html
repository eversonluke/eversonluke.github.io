In my university classes

"Hug the Angry Jarvis"

This is a basic game I wrote in Java for an assignment while in Dr. Jason Sawin's class during my junior year at St. Thomas. A player-versus-AI game of hide-and-seek in a playing arena that resembles a sort of Etch-a-Sketch, here are the basic rules of the game:

  • You begin in the top-left corner of the grid, and Jarvis begins in a random location
  • Every 800 milliseconds, Jarvis moves to an position adjacent to his current one
  • After every sixth move, Jarvis will lay an invisible homework trap adjacent to his current position
  • You are able to move to any adjacent tile to your current position, and provide inputs as often as you wish
  • Every position you step on becomes "visible", meaning that you can see Jarvis if/when he is there
  • Once Jarvis leaves a position, it will then become "invisible" again
  • If the player enters a position that holds a homework trap, he/she cannot move for 5 seconds
  • The player's objective is to observe how the board changes over time to gather clues regarding the location of Jarvis
  • Once the player enters the tile where Jarvis currently resides, the game is won

In my personal life

4-versus-4 Team Generator

Unrelated to any of my past classes but using the skills I gained from them, this is a simple 4-versus-4 team generator I wrote using the C programming language that my friends and I have used countless times while playing video games together since quarantine first started. Who even knows the total amount of time and the number of headaches this simple chunk of code has saved us-- here are a few of its features:

  • Uses the current time on the computer running the program as a seed to create pseudo-random values corresponding to how each player will be placed on a team
  • The user can specify pairs of players that he/she DOES wish to be placed on the same team
  • The user can specify pairs of players that he/she does NOT wish to be placed on the same team
  • To mix things up, there is a 1/100 chance each time that the program is run that a 5-versus-3 setup will be generated instead
  • Easily adjustable to meet the user's specific needs

This is easily my most used piece of code that I have written for use outside of my classes. If you are interested in trying out the code yourself, feel free to email me at the address provided in the "Resume" section!
